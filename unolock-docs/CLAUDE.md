# CLAUDE.md — UnoLock 프로젝트 안내

> 이 파일은 Claude Code가 프로젝트를 이해하기 위한 안내서입니다.
> 작업 전 반드시 이 문서와 `설계자료/` 폴더를 먼저 읽어주세요.

---

## 프로젝트 한 줄 정의

**교재만 올리면 AI가 학생 맞춤 미션을 내고, 과제를 완료한 만큼 스마트폰을 열어주는 학습 습관 형성 플랫폼.**

(앱 이름 'UnoLock'은 가칭이며 변경 가능)

---

## 만드는 사람에 대해

- 기획자(대표)는 **비전공자 수준**의 개발 지식을 가짐. 코드를 직접 쓰지 않고 Claude Code와 "바이브코딩"으로 진행.
- 따라서 **모든 설명은 쉽게**, 전문용어는 풀어서. 명령어는 복붙 가능하게 정확히.
- 기획자는 현직 과외 선생 (월 평균 매출 1,200만원). 교육 시장 현장 감각이 매우 뛰어남. 그 통찰을 신뢰할 것.

---

## 핵심 설계 철학 (반드시 지킬 것)

이 프로젝트는 기존 "폰 차단 앱"과 철학이 다릅니다. 아래 4개 기둥이 핵심입니다.

1. **교사 노동 0** — 교사는 교재(PDF·사진)만 올린다. AI가 문제를 자동 출제한다. 교사가 직접 문제 내게 하면 절대 안 됨 (도입 거부의 원인).

2. **미션은 시험이 아니라 리추얼** — 학생을 거르는 게 목적이 아니라, 매일 책상에 앉는 습관을 만드는 가벼운 트리거. 난이도는 의도적으로 낮게.

3. **8할 기준이되, 결국 통과시킨다** — 커트라인은 80%로 높게 잡되(학부모 명분), 틀리면 해설 후 비슷한 새 문제를 재출제해 결국 통과시킴(학원 이탈 리스크 0). 수학=재출제 루프, 영단어=뜻 알려주고 재시험.

4. **정보 비대칭** — 교사는 모든 데이터(틀린 횟수, 재시험 횟수)를 보지만, 학부모는 "통과 성공" 결과만 본다. 과정의 시행착오는 학부모에게 숨김 (학원 이미지 보호).

> 한 문장 요약: **UnoLock은 학생을 평가하는 도구가 아니라, 학원의 신뢰를 지켜주는 도구다.**

## 사용자 구조 (3종 + 권한 분리)

- **출제자** (학원 / 과외선생 / 직접학부모): 교재 업로드, 미션 설정, 전체 데이터 열람
- **학생**: 미션 수행, 보상 획득
- **학부모**: 보상 시간·앱 범위 설정(시간관리 권한), 성공 결과만 열람

**"직접학부모"** = 학원 없이 학부모가 직접 출제자 역할까지 하는 경우 (저학년 타겟).

---

## 보상 체계 (2단계 + 저축)

- **1단계 (중간 휴식)**: 과제 1개 완료마다 5~10분 적립. 짧게. 당겨쓰거나 모아둘 수 있음.
- **2단계 (최종 보상)**: 그날 **모든 학원의 모든 과제** 완료 시 → 학부모가 정한 시간까지 개방. 그날 한정.
- **휴식 지갑**: 1단계 휴식은 저축·이월 가능. 저축 한도는 학부모가 설정.
- 한 학생이 **여러 학원**에 등록 가능 → 과제가 출처별로 쌓이고 "오늘의 미션" 하나로 통합됨.
- 아이가 "지금 쓸까 / 모아둘까"를 선택 → 자기조절 능력 훈련이 핵심 가치.

---

## 계정 / 가입 흐름

- 학원이 먼저 가입 → 공간 생성 → 학생·학부모가 학원 검색해서 등록 신청 → 학원 승인.
- **자녀 증명**: 학생 등록 시 학부모 번호 입력 → 코드 발송(초기 SMS, 추후 카카오 알림톡) → 학부모 가입 시 코드 확인.
- 학원 이동: 학원이 인당 구독 끊음 → 데이터 3개월 보관 후 삭제.
- 직접학부모 전환: 구독 종료 시 자동 카톡 → 학부모가 직접 관리로 전환.

## 기술 방향

- **안드로이드 먼저** (iOS는 권한 우회 구멍 때문에 2차). 이유는 설계자료 기획서 참고.
- AI 출제: Claude API 사용. 비용은 미션당 약 10원 수준 (종량제). 단순 과제는 Haiku, 복잡하면 Sonnet으로 라우팅.
- 앱 사용 통계·휴식시간 보고 기능은 AI 불필요 (단순 집계).

---

## 개발 전략 — "설계도 먼저"

- 처음부터 전체 구조의 자리를 잡아두고, 1층부터 짓는다. 나중에 큰 기능(폰 잠금 등)이 깔끔하게 끼워지도록.
- 특히 **계정·권한 구조**가 가장 중요한 기초공사. 이걸 대충 하면 나중에 다 무너짐.

### 1차 MVP 범위 (지금 만들 것)

폰 잠금 기능을 **빼고**, 핵심 학습 루프만 먼저 만든다:

```
교재 업로드 → AI 출제 → 학생 문제 풀이 → 채점·재시험 → 통과 → 학부모 통과 알림
```

이것만으로 기획자의 실제 과외 학생에게 바로 테스트 가능. 폰 잠금·결제·리포트 자동화는 2차 이후.
